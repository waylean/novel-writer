# Chapter 1-11 Edit Analysis

## Important Limitation

No exact original drafts were found in the project folder. The folder is not a Git repository, and no `original`, `backup`, or old chapter files were present. Therefore this is not a line-by-line diff.

This analysis triangulates from:

- current chapter drafts 1-11
- approved chapter contracts in `outline/chapters.yaml`
- continuity reviews in `reviews/`
- later batch-style chapters 12-24 as a rough baseline for AI-generated continuation style

## Overall Editing Pattern

The revised chapter text is included in `chapters/ch001.txt` through `chapters/ch011.txt`.

The revisions appear to improve the draft in seven main ways:

1. **Causal logic is stronger.** Events now happen because Ning Zhao investigates, tests, models risk, or exploits constraints.
2. **Exposition is delayed and shortened.** Rules appear near the decision that needs them.
3. **Hooks are sharper than the default contract hooks.** Many endings upgrade from "the next event happens" to "a specific unresolved pressure appears."
4. **The protagonist is more consistent.** Ning Zhao reads as cautious, technical, and procedural rather than merely brave.
5. **The antagonist side is less stupid.** Black Reef Island has records, permissions, internal factions, and audit pressure.
6. **Scene texture is more concrete.** Smells, tools, screens, doors, wires, cuffs, and medical pain make the logic physical.
7. **Moral cost is preserved.** After taking the villain body, Ning Zhao still has to handle the original villain as a necessary risk, not as revenge.

## Author Editing Profile Inferred

### Keep

- Specific physical procedures before abstract explanation.
- Scene-level details that make systems tangible: folders, wires, badges, records, cuffs, doors, medical devices.
- Protagonist agency through verification, contingency planning, and exploiting institutional rules.
- Hooks that ask a precise next question.
- Antagonists and organizations that have audit trails, hierarchy, and self-protection.

### Avoid

- Long rule dumps.
- Easy protagonist success.
- Side characters who help too quickly.
- Villains who fail because they become stupid.
- Chapter endings that only say "and then the next event began."

### Future Drafting Requirements

- Every chapter needs a visible constraint.
- Every major protagonist action needs reason, test, risk, and fallback.
- Every rule should be introduced near the scene where it matters.
- Every chapter ending should be stronger than the default contract hook.
- Every victory should create a new problem.

## Skill Improvement Derived From This Case

The open-source skill should not stop at continuity checking. It should include a human-edit loop:

1. Detect whether true original drafts exist.
2. If true originals exist, perform a line-level comparison.
3. If true originals do not exist, triangulate from contracts, reviews, and unedited chapters.
4. Extract an author editing profile.
5. Feed that profile into future chapter contracts and drafts.

This is the step that reduces future manual revision.
